## Methodology check

The methodology introduced in 0.9.0 — see `skills/`:

- `skills/features.md` — feature authoring
- `skills/bugs.md` — bug authoring + TDD
- `skills/milestones.md` — `ROADMAP-<x.y.z>.md` per release, `done/` archive
- `skills/version.md` — `VERSION` + `.rpk/version` + ledger
- `skills/testing.md` — three test tiers
- `skills/logging.md` — debug / info / warn / error levels
- `skills/automerging.md` — manual + GitHub auto-merge
- `skills/audit.md` — this audit's reference

Held up across 4 milestones (0.9.0 → 0.12.0). Notable wins:

- TDD discipline: FEAT-211 and FEAT-212 each had a failing test
  pushed first, then the fix. The reproducing tests would have
  caught any regression.
- FEAT-214 (reconcile-on-wake) emerged from real-world friction
  (missed webhook on PR #2) and has been firing on every PR since.
- Roadmap shuffles (FEAT-213 0.11.0 → 0.12.0) tracked cleanly in
  both source and destination `ROADMAP-*.md` files.

No methodology debt identified.
